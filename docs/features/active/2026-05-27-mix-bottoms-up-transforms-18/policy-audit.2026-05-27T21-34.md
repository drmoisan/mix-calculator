## Executive Summary

This feature adds a pure-transform stage that builds three BottomsUp mix tables
(`mix_2_sku_bottomsup`, `mix_3_category_bottomsup`, `mix_4_customer_bottomsup`)
from tables the mix-decomposition pipeline already produces, and wires them into
`run_transforms` in `src/mix_pipeline_run.py`. The change is additive: no existing
table schema, column, or module behavior is modified, and no new CLI surface is
introduced.

The full Python toolchain (Black, Ruff, Pyright, Pytest with branch coverage) was
re-run by the reviewer against the changed files and passed cleanly in a single

criteria map to passing tests. The confidentiality hard constraint was verified by
