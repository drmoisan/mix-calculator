## Semantic Change (the one idea this child implements)

`required` is redefined from "must be present in the source to load" (input-presence) to "required OUTPUT column" â€” the output-identity set (one value column plus its dimension columns). `in_output` continues to mean "emitted in the final table (may be true without `required`)". The forward migration `2.0 -> 3.0` recomputes `required(3.0) = required(2.0) AND in_output(2.0)` so a column that was only an input-presence requirement but is not emitted is no longer a required-output column. The migration preserves `in_output` exactly, so emitted output is unchanged.
