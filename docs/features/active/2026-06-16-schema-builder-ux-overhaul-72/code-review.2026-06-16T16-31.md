## Executive Summary

This branch is a usability overhaul of the Schema Builder dialog (Issue #72, a follow-up to #50/#62). It delivers eleven acceptance criteria across resize behavior, the Identity, Derived, Columns, Key, Dedup, and Preview tabs. The change is Python-only: 20 production files and 12 test files (4147 insertions, 814 deletions across 51 files including docs/evidence). Reviewed evidence: the full branch diff against `main`, an independent re-run of the Black/Ruff/Pyright/Pytest toolchain, branch coverage (`artifacts/python/lcov.info` regenerated by the reviewer), and the feature folder QA-gate artifacts.

## Implementation Audit

### Python implementation audit

#### What changed well

- **D-3 scope discipline.** The `auto` dedup change in `_schema_model_specs.py` is minimal and surgical: add `"auto"` to `DEDUP_MODES` and narrow the discriminator-required guard from "any collapsing mode" to the explicit `{"collapse", "aggregate"}` set. The LE explicit `select_from`/discriminator behavior is untouched and has a dedicated regression test.
- **D-1 isolation.** Bracket handling is confined to `_schema_builder_derived_format.py` and the UI double-click slot in `_derived_formula_dialog.py`. No formula-engine source file is in the diff, so the `asteval` grammar is provably unchanged.
- **AC-9 real wiring.** `install_preview_refresh_handler` (`_schema_open_helpers.py:115`) connects a handler that calls `presenter.update_preview(...)`; it is invoked from the production `open_schema_builder` path (`_schema_wiring.py:157`). The integration test drives the real `QTabWidget.currentChanged` seam rather than calling `update_preview` directly.
- **Pure transform separation.** `apply_auto_dedup` is a side-effect-free pandas transform driven by column roles, with `TYPE_CHECKING`-guarded imports and a fail-fast `ValueError` when no measure columns exist.

#### Typing and API notes

- Pyright is clean (0 errors). No `Any` was introduced. `ColumnsTabViewProtocol` is a `@runtime_checkable typing.Protocol`, an appropriate seam for the Qt widget and a test fake. `DedupPolicy` remains a frozen-style dataclass enforcing its invariant in `__post_init__`. No public API was broken; `get_identity`/`set_identity` keep their `tuple[str, str, str]` contract.

#### Error handling and logging

- Errors are specific: `apply_auto_dedup` raises `ValueError` with a clear message; the Preview tab surfaces `SchemaValidationError` text and a "No source data available to preview" message rather than rendering nothing. No broad `except` and no `print` statements were added.
