## Strategy

CF2 introduces an explicit `ColumnSpec.located_by_name` boolean (default `False`) and decouples the loader's which-columns-to-keep set and `none`-dedup emitted column ORDER from the `required` flag. The binding constraint, documented in both CF1 findings, is that after AOP measures flip to `required: false`, the months/quarters become field-identical to the genuine presence-optional `YTG` (`role=measure, required=False, in_output=True, numeric=True`); the only signal that distinguished the trailing `YTG`/`KEY` from the always-present measures was the pre-3.0 `required` flag. `located_by_name` restores that signal independently of `required`.

Safe sequencing (the central risk control):
1. Land the schema-model field + serialization + migration seeding (Phase 1).
2. Land the loader decouple keyed off `located_by_name`, with order reproducing `load_aop` flag-independently (Phase 2).
3. Prove AOP + LE parity UNCHANGED with the loader decouple in place but BEFORE flipping any AOP `required` flag, including a simulated-flip order-independence test (Phase 3).
4. Only then set `located_by_name` on LE and minimize `default_aop` `required` flags, re-proving parity after each (Phases 4–5).

Order facts the loader must reproduce (the oracle, from `_load_aop_helpers.SOURCE_COLUMNS`/`EXPECTED_COLUMNS` and the parity test):
- AOP `none`-path emitted order is the resolution order: `EXPECTED_COLUMNS` (Customer, SKU Descripiton, SKU #, Customer Master, Type, Jan..Dec, YTD, Q1..Q4, Super Category, PPG) FIRST, then the by-name-located optionals appended in `KEY`-then-`YTG` order. Final: `Customer..Type, Jan..Dec, YTD, Q1..Q4, Super Category, PPG, YTG, KEY`.
- The declared `columns` order in `default_aop.schema.json` is NOT the emitted order (KEY is declared first; YTG/Super Category/PPG ordering differs). The keep-list/order rule must therefore separate `located_by_name` columns (appended last, in schema order) from non-located columns (kept in `EXPECTED_COLUMNS`/resolution order), not reorder by declared index.

Fail-closed evidence rule: each phase records baseline, regression, and QA artifacts under `docs/features/active/etl-loader-output-required-and-drop-74/evidence/<kind>/`. Missing required artifact ⇒ verdict BLOCKED/INCOMPLETE, never PASS.

Note on AOP source version: `default_aop.schema.json` is `version: "2.0"`, so the `2.0 -> 3.0` migration in `schema_serialization._object_to_column` runs on load. The migration must seed `located_by_name` for pre-3.0 sources as `located_by_name = not required(2.0)` (the columns that were `required=False` under the old loader — AOP `KEY`/`YTG`, LE `KEY`/`YTD/YTG`). For 3.0-or-later sources the parsed value passes through unchanged. Item 4 sets `located_by_name` explicitly in the AOP JSON so it does not rely on the migration after the `required` flip.
