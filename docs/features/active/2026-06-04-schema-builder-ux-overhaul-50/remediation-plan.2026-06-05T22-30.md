- B1 (primary) — guard discovery against blank/empty path or sheet:
  - Implementation: `[P1-T1]` (presenter guard), `[P1-T2]` (optional wiring short-circuit).
  - Wiring-level integration test (carries B1 proof): `[P2-T1]` (no file selected) and `[P2-T2]` (file selected, no worksheet). `[P2-T1]` is the primary wiring-level integration test for B1.
- B2 (coupled) — non-crashing unknown/blank-sheet reader error contract:
  - Implementation: `[P1-T3]` (reader raises `ValueError` for an absent sheet) and `[P1-T4]` (reader docstring updated to match the chosen contract).
  - Unit test (carries B2 proof): `[P2-T3]` (reader raises `ValueError` for an unknown sheet) and `[P2-T4]` (presenter routes that `ValueError` to `show_error` with no crash).

Chosen B2 contract: `WorkbookReader.read_sheet_preview` raises `ValueError` for an absent/unknown sheet. Rationale: it aligns the reader with the presenter's existing reader-error policy used by `on_file_selected` / `on_render_tab` / `on_schema_discovery` (all catch only `ValueError`), so no new exception type must be added to the presenter and the existing `test_on_schema_discovery_reader_value_error_routes_to_show_error` contract is preserved.
