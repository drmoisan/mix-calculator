## Scope and overrides

- Scope 1B (user-approved): route the DEFAULT GUI AOP import through `SchemaLoader(default_aop)`; retire the legacy arithmetic-validating `load_aop` for the IMPORT path only.
- Decision 2 (user-approved): remove the derived blank-total fill by clearing `fill_rules` in the default AOP schema (blank totals coerce to 0).
- Research override (AC-7): the research's "read at header=2" finding predates a base change. The branch is rebased onto `main` post-PR-#56, so `src/_header_detection.py` and `detect_header_row` ARE available. `import_aop` MUST resolve the AOP header row via `detect_header_row`, not hardcode `header=2`.
- `min_match` decision: the legacy `src/load_aop.py` calls `detect_header_row(..., min_match=17)` against `EXPECTED_COLUMNS` (24 required tokens = SOURCE_COLUMNS minus `{KEY, YTG}`; `load_aop` comments "17 of the 24"). The schema-driven path MUST derive `expected_tokens` from the loaded `default_aop` schema's column canonical names (normalized via `src.etl_columns.normalize_name`), excluding the optional `KEY` and `YTG` columns to match the legacy required-token set, and reuse `min_match=17` for behavioral parity with `load_aop` header detection.

## Non-goals (do not modify)

- The legacy CLI AOP path is unchanged: `src/load_aop.py` `main`, `validate_aop`, and `build_per_row_checks` are not modified and remain callable by `mix_pipeline._import_sources`. `coerce_numeric` and `clean_label_sentinels` in `src/_load_aop_helpers.py` remain (consumed by `SchemaLoader._coerce_and_clean`).
- No change to the LE (`import_le`) or SKU_LU (`import_skulu`) import paths or their loaders.
- No new dependency; no `SCHEMA_FORMAT_VERSION` bump (data-only schema edits within the existing `2.0` shape).
- Do NOT include computed workbook aggregate figures (confidential) anywhere in plan, code, tests, or evidence.

## Evidence and per-batch budget

- All evidence artifacts MUST be written under `docs/features/active/2026-06-08-aop-import-schema-driven-58/evidence/<kind>/` per `evidence-and-timestamp-conventions`. Non-canonical `artifacts/...` evidence paths are rejected.
- Per-batch budget: at most 3 production files and 3 test files per implementation batch. The three production files are sequenced across two implementation phases (Phase 1: schema JSON + `schema_loader.py`; Phase 2: `pipeline_service.py`), each within budget.

Restart from formatting on any change. >= 85% line / >= 75% branch; no regression on changed lines; no new suppressions; 500-line file cap; docstrings per self-explanatory-code-commenting; professional tone.
