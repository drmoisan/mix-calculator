## Design Direction (planner-owned, follows the non-binding inputs preference)

Single seam for "available schemas" = union of bundled defaults and user-saved schemas,
with user-saved entries overriding bundled entries of the same name. The seam lives in
`SchemaRegistry` so that `list_schemas()` and `load()` become union-aware. Because
`find_best_match_in_registry` (`src/schema_matching.py:430`) and
`SchemaService.list_schema_names`/`load_schema` (`src/gui/services/schema_service.py`)
already delegate to `registry.list_schemas()`/`registry.load()`, making the registry
union-aware fixes Gap 1 (listing), Gap 3 (matching), and the load-by-name path
(R-AC-5) at one source. No first-run "copy bundled files to `%APPDATA%`" approach is
used; no launch-time disk side effects are added.

Dropdown population (Gap 2) is wired in a new small composition-root module
`src/gui/_schema_list_wiring.py` (following the `_run_wiring.py` / `_key_mismatch_seam.py`
precedent) so `src/gui/app.py` does not grow past the 500-line cap (it was 499 at #48
entry; current size is 494 lines, headroom is small and must not be consumed in place).
The new module calls `view.set_schema_list(service.list_schema_names())` on each of the
three source widgets at startup.

Additive-only constraints: `load_bundled_default` is retained unchanged; the known-file
loaders (`import_le`/`import_aop`/`import_skulu`) and user-registry persistence semantics
(`save`, suffix handling, `resolve_registry_dir`) are not modified.
